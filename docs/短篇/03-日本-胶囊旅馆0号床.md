# 胶囊旅馆 0 号床

## 故事正文

森下胶囊旅馆开在车站背后，招牌很小，走廊却深得像没有尽头。旅馆一共有五层，每层二十四个胶囊床位。系统编号从 1 到 120，没有 0 号床。

游客许成在这里住了一晚后失踪。他来自外地，独自旅行，入住记录显示床位为 418。前台说他第二天没有退房，清洁员打开胶囊时，里面只有一只耳机和半瓶水。监控拍到他凌晨两点四十分从洗手间返回，之后再也没有出来。

调查员检查电子门锁。418 胶囊在两点四十一分关闭，三点零三分又打开一次，开锁卡号为 000。旅馆经理解释，000 是系统维护卡，不对应实体房卡，只在后台测试时出现。

更怪的是，入住系统每晚三点都会自动生成一条记录：床位 0，姓名空白，证件号空白，入住状态为“已躺下”。这条记录第二天清晨会自动消失，只有备份日志能看到。

许成的手机云端同步了几段录音。第一段是他给朋友发语音：“这里隔音太差，旁边一直有人翻身。”朋友回复说，418 左右两边分别是墙体和消防柜，不可能有人睡在旁边。

第二段录音里，许成压低声音说：“我听见有人从头顶爬过去。”随后是指甲划过塑料板的声音，缓慢、规律，像有人沿着胶囊外壳寻找入口。

调查员进入 418。胶囊很窄，天花板有一块检修盖。打开后，上方不是管线，而是一条低矮夹层，刚好能容一个成年人匍匐通过。夹层里铺着灰尘，灰尘中有拖拽痕迹，通向走廊尽头的布草间。

布草间墙后还有一排封死的旧胶囊。旅馆前身是廉价网吧，火灾整改时改建成胶囊旅馆。原设计图上，靠近电梯井的位置确实有一个临时休息舱，编号为 0，供夜班管理员使用。后来因为通风问题被封闭。

七年前，一名夜班管理员死在 0 号舱里。官方说是突发心梗。可旧员工聊天记录显示，他死亡前连续一周投诉“客人半夜不退房”，还说系统里总有空白入住记录。他最后一条消息是：“0 号床有人，但我看不见脸。”

许成失踪前，在旅馆评价网站留下草稿。他写道，睡到半夜时，胶囊舱门外的蓝灯变成红灯，耳边有广播提示：“请客人向内侧移动，为 0 号床让出通道。”他以为是做梦，直到有人从外面轻轻敲他的舱门，问他：“你是今晚的替床吗？”

警方拆开封闭墙体，找到 0 号舱。舱内没有尸体，只有许成的外套、身份证和一排旧房卡。每张房卡都没有号码，卡面写着不同失踪者的姓名。最早一张属于七年前死亡的管理员。

技术员恢复系统备份，发现 0 号床记录并不是最近才出现。它从旅馆开业第一天就存在，只是后台界面隐藏。每次有人投诉夜间异响或申请换床，系统都会把他标记为“可替换客人”。

经理终于承认，当年改建时他们没有彻底拆除旧夹层。有人利用夹层偷窃客人物品，旅馆为了名声选择私了。后来偷窃者死在夹层里，尸体一直没找到。

许成仍然失踪。0 号舱被拆除后，旅馆停业。调查员离开前，前台打印机突然吐出一张入住单。

床位：0。

姓名：许成。

状态：已躺下。

## 游戏化要点

- **核心怪谈**：胶囊旅馆系统每天生成不存在的 0 号床。
- **真相方向**：封闭夹层、旅馆隐瞒事故、系统隐藏记录和失踪案相互串联。
- **关键场景**：418 胶囊、夹层通道、布草间、0 号旧舱、前台后台。
- **关键线索**：000 开锁卡、自动入住记录、录音、检修盖、旧设计图、无号码房卡。
- **检索关键词**：0号床、已躺下、000、夹层、替床、布草间。
- **复盘问题**：许成如何从封闭胶囊中失踪？0 号床记录为什么会自动生成？
